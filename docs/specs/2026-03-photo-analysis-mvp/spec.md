# Feature Spec: Photo Analysis MVP

## 1. 背景

- IroMap の主価値を「写真の色構造分析」に置く
- RGB可視化は技術MVPとして維持しつつ、分析価値を機能MVPとして明確化する

## 2. スコープ

### In Scope

- Lab a-b scatter を表示する
- Hue histogram を表示する
- Saturation histogram を表示する
- Color area ratio（主要色の面積比）を表示する
- Lab scatter / Hue histogram / Saturation histogram に軸ラベルを表示する
- Slice表示を含む主要グラフで、共通の軸表示フレーム実装を利用する
- キューブへの画像プロット操作時に、アップロードUIをキューブ近傍へ配置する

### Out of Scope

- RGB cube の描画基盤実装
- 診断文章の自動生成
- Cluster map / DeltaE heatmap / Saliency map

## 3. 利用者 / ステークホルダー

- 写真の色傾向を把握したい撮影者
- カラー設計を分析したいデザイナー

## 4. ユーザーストーリー

- ユーザーとして、写真の色分布をLab平面で確認したい。なぜなら色クラスタの偏りを把握したいから
- ユーザーとして、色相と彩度の分布をヒストグラムで確認したい。なぜなら写真の印象を定量的に理解したいから
- ユーザーとして、主要色の面積比を確認したい。なぜなら構図上の色バランスを把握したいから

## 5. 機能要求

### FR-1: Lab a-b scatter を表示できる

#### 受け入れ条件

- x=a, y=b の2次元平面に色サンプルをプロットできる
- 色クラスタの偏りを視認できる

### FR-2: Hue histogram を表示できる

#### 受け入れ条件

- 色相の分布をビン集計して表示できる
- 優勢な色相帯を視認できる

### FR-3: Saturation histogram を表示できる

#### 受け入れ条件

- 彩度の分布をビン集計して表示できる
- 低彩度画像かどうかを判別できる

### FR-3.5: 主要グラフで軸情報を明示できる

#### 受け入れ条件

- Lab a-b scatter で X=a*, Y=b* の軸表示が確認できる
- Hue histogram で X=Hue, Y=Count の軸表示が確認できる
- Saturation histogram で X=Saturation, Y=Count の軸表示が確認できる

### FR-4: Color area ratio を表示できる

#### 受け入れ条件

- 主要色ごとの比率をパーセント表示できる
- 合計が100%になる（丸め誤差許容）

### FR-5: 画像アップロード操作をキューブ近傍で完結できる

#### 受け入れ条件

- 画像アップロードUIは、RGBキューブの操作コンテキストから離れない位置に配置される
- アップロード後のキューブ上プロット結果を、同一視界で確認できる

## 6. 非機能要求

- NFR-1: 1920x1080 相当画像を対象に、分析表示開始まで2秒以内（開発環境の目安）
- NFR-2: 同一入力画像に対して分析結果が再現可能である

## 7. 業務ルール / 制約

- 画像1枚に対して分析結果1セットを生成する
- 色空間変換は sRGB 前提で扱う

## 8. エッジケース

- 単色画像でもグラフが崩れない
- ほぼ無彩色画像でもヒストグラムが正しく表示される

## 9. 成功指標

- ユーザーが「主要色」「色相傾向」「彩度傾向」を1画面で説明できる
- 主要色面積比の認知が主観評価と大きく乖離しない

## 10. 未決事項

- 面積比の色クラスタ数（固定値か自動推定か）
- ヒストグラムのビン数の初期値
