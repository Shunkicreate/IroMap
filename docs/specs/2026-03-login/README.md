# Login Spec Set README

`docs/specs/2026-03-login/` は「ログイン機能」の仕様セットです。

## 意図
- ログイン機能の要求、設計、実装計画、検証観点を1箇所に集約する
- 要求 ID（`FR-*`）を軸にトレーサビリティを確保する

## ファイルの意味
- `spec.md`: 背景、スコープ、FR/NFR、未決事項
- `design.md`: 実装方針、API、例外、セキュリティ、可観測性
- `tasks.md`: 実装/テスト/運用タスク（`T-*`）
- `test-cases.md`: 正常系/異常系の受け入れ観点
- `notes.md`: 検討中メモ

## 書き方フォーマット
```md
# Feature Spec: ログイン
## 5. 機能要求
### FR-1
...
#### 受け入れ条件
- ...
```

```md
# Tasks: ログイン
- [ ] T-001 ...（FR-1）
  - 完了条件: ...
```

## 記述ルールの意味
- `FR-*`: ユーザー価値に直結する要求
- `受け入れ条件`: 実装完了の判定基準
- `T-*`: 実装粒度の作業単位（検証可能であること）
