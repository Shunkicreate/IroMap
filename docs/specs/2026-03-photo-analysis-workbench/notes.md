# Notes: Photo Analysis Workbench

- この spec は `2026-03-photo-analysis-mvp` の後続段階として扱う
- MVP の主眼は `相互参照 / 局所選択 / 2 件比較 / 指標表コピー / L* histogram` に置く
- `Lab a-b scatter` は主表示から外すが、完全削除ではなく補助分析として残す余地を残す
- `Neutral Distance` は MVP では `C*` と同義で扱い、UI 上の名称だけ後続で調整可能にする
- 履歴、選択保存、注釈メモは P1 以降の状態設計だけ先に確保する

## 詳細化で足りていない論点

- 状態モデルの粒度
  - 現在は `target / selection / compare` の存在だけ定義しており、単一 source of truth と派生 state の境界が未確定
- 選択モデルの正規化
  - 画像矩形、3D hover、slice hover、将来のブラシ選択をどこまで共通の `sampleIds` モデルへ寄せるか未確定
- 比較モデルの単位
  - `画像比較`、`スナップショット比較`、`選択比較` の優先順位と ID モデルが未確定
- 指標仕様の厳密化
  - 各統計量の計算対象、欠損時の扱い、小数桁、表示順、差分計算式が未確定
- histogram 仕様
  - bin 数、bin 境界、0 件 bin の扱い、重ね表示ルール、コピー時フォーマットが未確定
- UI インタラクション
  - hover と click の責務、選択解除、複数選択禁止ルール、比較中の操作優先順位が未確定
- 性能設計
  - サンプル数上限、再計算のトリガー、重い計算の memoization / worker 分離要否が未確定
- テスト分解
  - domain test、UI test、e2e test の責務分離が未確定

## 設計詳細化の進め方

### Step 1: 設計対象を 7 つの論点に分割する

- A. 用語と境界
- B. 状態モデル
- C. 選択・相互ハイライト
- D. 指標・histogram・コピー仕様
- E. 比較・差分仕様
- F. UI レイアウト・操作フロー
- G. 性能・テスト・段階導入

### Step 2: 各論点で「決めること」を先に固定する

- A. 用語と境界
  - `target`, `snapshot`, `selection`, `comparison` の意味を 1 つに固定する
- B. 状態モデル
  - 永続 state、画面 state、派生 state を分ける
- C. 選択・相互ハイライト
  - hover と selected を別状態として持つかを決める
  - 画像起点選択と色空間起点選択の正規化ルールを決める
- D. 指標・histogram・コピー仕様
  - 各指標の計算式、丸め、小数桁、欠損ルール、出力列順を決める
- E. 比較・差分仕様
  - 何を baseline とし、差分の向きと単位をどう固定するかを決める
- F. UI レイアウト・操作フロー
  - 上段 / 下段構成、panel responsibilities、操作導線、空状態を決める
- G. 性能・テスト・段階導入
  - どこまでを P0 実装に含め、どこからを P1 へ逃がすかを決める

### Step 3: 先に domain 設計を固め、その後で UI 設計へ降ろす

- 先に固める
  - データ構造
  - 計算責務
  - 依存方向
  - 欠損 / 例外ルール
- 後で固める
  - レイアウト
  - 画面内の panel 構成
  - 操作フロー
  - コピー UI

### Step 4: 1 論点ごとに成果物を増やす

- `spec.md`
  - 受け入れ条件の厳密化
- `design.md`
  - 状態遷移、データ構造、計算責務、シーケンスを追加
- `notes.md`
  - 未決事項と代替案比較を残す
- `tasks.md`
  - 実装順が見える粒度まで分解する
- `test-cases.md`
  - 指標仕様や操作仕様に対応する観点を追加する

## 具体的な詳細化順序

1. 用語集と比較単位を固定する
2. 状態モデルを厳密化する
3. 選択と相互ハイライトのイベント仕様を定義する
4. 指標表と histogram の計算仕様を定義する
5. 比較と差分の baseline ルールを定義する
6. UI レイアウトと panel ごとの責務を定義する
7. 性能制約とテスト戦略を定義する
8. 最後に tasks と test-cases を再分解する

## この計画での進め方

- まず次のターンでは `状態モデル + 選択モデル + 比較モデル` を詳細化する
- その次に `指標仕様 + histogram 仕様 + コピー仕様` を詳細化する
- 最後に `UI フロー + 性能 + テスト戦略` を詰める

この順にすると、UI を先に決めて状態が破綻するのを避けられる。
