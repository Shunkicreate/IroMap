# Feature Spec: Photo Analysis Workbench

## 1. 背景

- IroMap を「色空間を見せるデモ」から、写真現像判断に使える分析ワークベンチへ進める
- 現状の photo analysis は単一画像の概観把握が中心であり、画像上の位置、3D 色空間、スライス、指標表、比較導線が分断されている
- 写真用途では、全体傾向だけでなく、主役、背景、白、暗部などの局所を分けて比較できることが重要である

## 2. スコープ

### In Scope

- 画像プレビュー、3D 色空間、スライスを同時表示する分析レイアウトを提供する
- 画像上の点や領域と、3D 色空間 / スライス上の点群を相互ハイライトできる
- 矩形選択を起点に局所領域を再集計できる
- 複数画像を比較対象として保持し、全体または選択領域単位で比較できる
- 写真現像に効く最小指標セットを表形式で表示し、Markdown Table / CSV / TSV でコピーできる
- L* histogram を追加し、少なくとも L* bin データをコピーできる
- 差分表示の最小セットとして `ΔL* / Δa* / Δb* / ΔC* / ΔE` を表示できる

### Out of Scope

- ブラシ選択
- 色域ベース選択
- 自動クラスタ抽出とクラスタ比較
- 色相エントロピーなどの高度統計
- 自動コメント生成
- 履歴への自由記述メモ保存

## 3. 利用者 / ステークホルダー

- 写真現像の判断材料を増やしたい撮影者
- 色の分布と被写体分離を検証したいデザイナー
- 既存の RGB cube / slice / inspector 導線を統合したいプロダクト開発者

## 4. ユーザーストーリー

- ユーザーとして、色空間で選んだ点が画像のどこにあるか見たい。なぜなら、色の意味を被写体上で確認したいから
- ユーザーとして、画像上で選んだ領域が色空間のどこにあるか見たい。なぜなら、主役や背景の分布を分けて理解したいから
- ユーザーとして、複数画像や現像前後を並べて比較したい。なぜなら、調整の効果を定量と見た目の両方で確認したいから
- ユーザーとして、分析結果を表としてコピーしたい。なぜなら、記録や外部メモへ再利用したいから

## 5. 機能要求

### FR-1: 分析ワークベンチの主表示を 3 視点で常時確認できる

#### 受け入れ条件

- デスクトップ幅では、上段に `画像プレビュー / 3D 色空間 / スライス + インスペクタ` の 3 領域が同時表示される
- 下段に `指標表 / ヒストグラム / 比較 / 履歴系` の補助領域を配置できる
- 既存の Lab a-b scatter は主表示から外し、分析の中心を 3D 色空間とスライスへ移す

### FR-2: 画像上の位置と色空間上の位置を相互参照できる

#### 受け入れ条件

- 画像プレビュー上の sample point または hover 対象について、少なくとも `位置 / RGB / HSL / Lab` を確認できる
- hover 状態と確定選択状態は区別され、hover の移動で確定選択が破壊されない
- 3D 色空間またはスライスで選択した点群に対応する画像領域をハイライト表示できる
- 画像上で選択した領域に対応する点群を 3D 色空間とスライス上で強調表示できる

### FR-3: 局所選択を起点に再集計できる

#### 受け入れ条件

- 最低限の選択手段として矩形選択を提供する
- 選択対象は `画像全体` と `選択領域` を切り替えて分析できる
- 選択領域に対して、指標表とヒストグラムが再集計される
- 選択領域の面積比率を補助情報として表示できる

### FR-4: 複数画像を比較し、差分を表示できる

#### 受け入れ条件

- 少なくとも 2 件の画像または解析スナップショットを比較対象として保持できる
- MVP の比較ペアは `baseline 1 件 + compare 1 件` の 2 件固定とする
- 比較単位として `画像全体` と `選択領域` を扱える
- 比較結果として `平均値比較 / ヒストグラム比較 / 3D 点群の重ね表示 / 差分値表示` を提供する
- 差分値表示には少なくとも `ΔL* / Δa* / Δb* / ΔC* / ΔE` を含む

### FR-5: 写真現像向けの最小指標表を表示できる

#### 受け入れ条件

- 指標表には少なくとも以下を含む
  - `L* mean / L* stddev / L* p95`
  - `a* mean / b* mean`
  - `C* mean / C* p95`
  - `Neutral Distance mean`
  - `Highlight b* mean / Highlight Neutral Distance mean`
  - `Selection A-B ΔE`
- 各行は `区分 / 指標 / 値 / 意味` を確認できる
- MVP での表示精度は、統計値を小数第 2 位まで表示する
- 値が算出不能な場合は `N/A` を表示し、空文字や 0 に置き換えない
- 比較モードでは、基準対象との差分列を追加表示できる

### FR-6: 指標表とヒストグラムをコピーできる

#### 受け入れ条件

- 指標表は `Markdown Table / CSV / TSV` の 3 形式でコピーできる
- デフォルトのコピー形式は `Markdown Table` とする
- ヒストグラムは各 bin について `start / end / count / ratio` をコピーできる
- 指標表コピーの列順は `group, key, label, value, unit, delta, description` を基準とする
- histogram コピーの列順は `metric, binIndex, start, end, count, ratio` を基準とする
- コピー成功時に利用者へ明示的なフィードバックを表示できる

### FR-7: L* を中心としたヒストグラム表示を提供できる

#### 受け入れ条件

- `L* histogram` を表示できる
- MVP での `L* histogram` は 20 bins の固定幅集計とする
- P0 では `L* histogram` を必須とし、`Hue / Saturation / C* or Neutral Distance` は既存導線と併存または段階追加できる
- ヒストグラム比較時は、対象間の分布差を視認できる

### FR-8: P1 として履歴・選択保存へ拡張できる

#### 受け入れ条件

- 今回の設計は、将来的に `読み込んだ画像履歴 / 解析スナップショット保存 / 選択領域保存 / メモ付与` を追加できる構造である
- MVP では最低限、比較対象や選択状態を UI 状態として安全に保持できる

## 6. 非機能要求

- NFR-1: 1920x1080 相当画像で、単一対象の再集計表示開始まで 2 秒以内を目安とする
- NFR-2: 同一画像、同一選択条件、同一サンプリング条件に対して再現可能な分析結果を返す
- NFR-3: 画像データと分析結果はクライアント内処理を基本とし、外部送信しない
- NFR-4: デスクトップ横長レイアウトを優先しつつ、狭幅でも主要導線が崩れない

## 7. 業務ルール / 制約

- 色空間変換は sRGB 前提で扱う
- Neutral Distance は `sqrt(a^2 + b^2)` とし、MVP では `C*` と同義に扱う
- Highlight Neutrality は `L* > 80` の画素群で集計する
- 比較対象は同時に多数保持せず、MVP では 2 件比較を優先する
- サンプルモードは将来 `全画素 / 間引き / 選択領域のみ` を持てる設計にするが、MVP では既存の間引き戦略を再利用してよい
- `hover` は一時状態、`selection` は明示操作で確定する状態として扱う
- `comparison` の差分方向は常に `compare - baseline` とする
- `L* histogram` の bin 境界は `0 <= L* <= 100` の等幅分割とする

## 8. エッジケース

- 単色画像でも選択、相互ハイライト、ヒストグラム表示が破綻しない
- ほぼ無彩色画像でも Neutral Distance / C* が 0 近傍として安定表示される
- Highlight 条件に該当画素がない場合は、該当指標を `N/A` として扱う
- 選択領域が極小または空の場合は、再集計や比較を実行せず空状態または案内を表示する
- 画像サイズやアスペクト比が異なっても、比較 UI が誤解を招く座標対応をしない

## 9. 成功指標

- ユーザーが「どの色が画像のどこにあり、どの程度分離しているか」を 1 画面で説明できる
- 現像前後比較で、見た目の差と数値差分を同時に確認できる
- 表コピー機能により、分析結果を外部メモへ再利用できる

## 10. 未決事項

- 矩形選択の後にブラシ選択をどのタイミングで追加するか
- 3D 色空間とスライスでの選択操作を共通モデルへ統一するか、段階導入するか
- 比較対象の保持単位を `画像` と `解析スナップショット` のどちらで先に固定するか
