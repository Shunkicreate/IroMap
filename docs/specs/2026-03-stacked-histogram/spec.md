# Feature Spec: Stacked Histogram for Photo Analysis

## 1. 背景

- 現在の Hue / Saturation ヒストグラムは単軸の件数表示で、分布の内訳（何がその山を作っているか）が分かりにくい
- 写真の印象把握には、単純な頻度だけでなく「どの属性が重なってその頻度を構成しているか」の可視化が必要
- 特に彩度分布について、Hue または Lightness ごとの層構造を見たいという要望がある

## 2. スコープ

### In Scope

- Saturation histogram を第2軸カテゴリで縦積み（stacked）表示できる
- Hue histogram を第2軸カテゴリで縦積み（stacked）表示できる
- 第2軸カテゴリを切り替えられる（例: Hue / Lightness / Saturation）
- 凡例とホバー情報で、各スタック区間の意味と件数を確認できる
- 既存の単純ヒストグラム表示へ戻せる（比較用）

### Out of Scope

- 分析結果に対する自動診断文生成
- 3D可視化（RGB cube / Slice）側の新規連携
- クラスタリングアルゴリズムの刷新

## 3. 利用者 / ステークホルダー

- 写真の色設計を分析したい撮影者・デザイナー
- レタッチ前後の色傾向変化を定量比較したいユーザー

## 4. ユーザーストーリー

- ユーザーとして、彩度ヒストグラムを Hue または Lightness 別に積層表示したい。なぜなら同じ彩度帯でも色相や明度の内訳差を把握したいから
- ユーザーとして、色相ヒストグラムを Saturation または Lightness 別に積層表示したい。なぜなら優勢色相の質感（鮮やかさ・明るさ）を判断したいから
- ユーザーとして、単純表示と積層表示を切り替えたい。なぜなら全体傾向と内訳傾向を使い分けたいから

## 5. 機能要求

### FR-1: Saturation histogram を縦積み表示できる

#### 受け入れ条件

- x軸を Saturation bin とし、各binを複数カテゴリで縦積み表示できる
- カテゴリ種別として少なくとも `Hue` と `Lightness` を選択できる
- 各binの合計値は、同binの総ピクセル数と一致する
- 単純表示（非スタック）と同一入力で総数整合が取れる

### FR-2: Hue histogram を縦積み表示できる

#### 受け入れ条件

- x軸を Hue bin とし、各binを複数カテゴリで縦積み表示できる
- カテゴリ種別として少なくとも `Saturation` と `Lightness` を選択できる
- 各binの合計値は、同binの総ピクセル数と一致する
- 単純表示（非スタック）と同一入力で総数整合が取れる

### FR-3: スタック軸と表示モードを切り替えできる

#### 受け入れ条件

- ヒストグラムごとに `表示モード（Simple / Stacked）` を切り替えできる
- Saturation histogram では、Stacked時に `Hue` と `Lightness` を相互に切り替えできる
- Hue histogram では、Stacked時に `Saturation` と `Lightness` を相互に切り替えできる
- 2つのヒストグラムは独立して設定できる（片方のみ切替、別軸選択が可能）
- 切り替え操作後にグラフが即時再描画される（再アップロード不要）

### FR-4: 凡例・ツールチップで内訳を読める

#### 受け入れ条件

- 各スタックカテゴリに対応する凡例を表示できる
- ホバー時に `x軸bin範囲 / スタックカテゴリ / count / 構成比` を確認できる
- カテゴリ色と凡例色が一貫している

### FR-5: ビン定義とカテゴリ定義が明示される

#### 受け入れ条件

- Hue/Saturation/Lightness の各ビン分割数を仕様上で固定または設定可能として定義する
- 仕様で定義したビン境界に従って集計される
- 端値（例: hue=360, saturation=1.0）の取り扱いルールが明示される

## 6. 非機能要求

- NFR-1: 1920x1080 相当画像で、モード切替後の再描画は 200ms 以内を目安とする（再分析不要の場合）
- NFR-2: 同一画像・同一ビン定義・同一モードで再現可能な集計結果を返す
- NFR-3: 可読性を担保するため、カテゴリ数増加時は凡例・配色コントラスト基準を満たす

## 7. 業務ルール / 制約

- 基本色空間は sRGB を前提とし、集計指標は HSV/HSL/Lab のうち仕様で定義したものを使用する
- 既存 `photo-analysis-mvp` の結果整合性を壊さない（総ピクセル数、既存ヒストグラム総数）
- スタックカテゴリの並び順は固定ルールを持つ（値昇順など）

## 8. エッジケース

- ほぼ無彩色画像でも Hue 由来カテゴリの表示が破綻しない
- 単色画像でも1カテゴリ偏重のスタックとして正しく表示される
- サンプル数が少ない画像でも、0件カテゴリを含む凡例表示が崩れない

## 9. 成功指標

- ユーザーが「同じピークでも内訳が異なる」ケースを説明できる
- 単純ヒストグラムのみの場合より、色傾向の説明精度が向上したと評価される
- モード切替利用率（Simple ⇄ Stacked）が一定以上あり、機能が実利用される

## 10. 未決事項

- Lightness は HSL-L と Lab-L* のどちらを採用するか
- デフォルトのスタック軸（Saturation側は Hue / Hue側は Saturation を初期値とするか）
- カテゴリ数の上限と配色戦略（固定パレット / 動的生成）
- UI配置: ヒストグラム個別設定にするか、Photo Analysis全体の共通設定にするか
