# Feature Spec: ユーザー登録

## 1. 背景
- 新規ユーザーが安全にサービス利用を開始できる登録導線が必要
- 今後の認証機能拡張の前提として、登録フローを先に固める必要がある
- 未整備のままだと不正登録や誤入力によるサポート負荷が増える

## 2. スコープ
### In Scope
- メールアドレスとパスワードで新規登録できる
- 確認メール送信を行う
- メール認証完了前は `pending` 状態として扱う

### Out of Scope
- SNS ログイン
- 管理者による手動登録

## 3. 利用者 / ステークホルダー
- 新規ユーザー
- CS
- セキュリティ担当

## 4. ユーザーストーリー
- ユーザーとして、メールアドレスで登録したい。なぜならサービスを利用開始したいから。
- ユーザーとして、誤入力したメールアドレスでは登録完了したくない。なぜなら通知を受け取れなくなるから。

## 5. 機能要求
### FR-1
ユーザーはメールアドレスとパスワードを入力して登録申請できる。

#### 受け入れ条件
- 正しいメールアドレス形式のみ受け付ける
- パスワードは 12 文字以上である
- 登録成功時、仮登録状態のユーザーが作成される

### FR-2
システムは確認メールを送信する。

#### 受け入れ条件
- メールには 30 分有効な確認リンクが含まれる
- 同一メールアドレスへの再送は 60 秒に 1 回まで

## 6. 非機能要求
- セキュリティ: メール存在有無を推測しにくいエラーメッセージ
- 性能: 登録 API の p95 応答時間は 300ms 以下（メール送信非同期前提）
- 可観測性: 成功率、送信失敗率、再送回数を計測する

## 7. 業務ルール / 制約
- メール認証完了前はログイン不可
- 同一メールアドレスの重複登録不可

## 8. エッジケース
- 確認リンク期限切れ
- 連打
- 既存退会ユーザーの再登録
- メール送信基盤障害

## 9. 成功指標
- 登録完了率
- メール認証完了率
- サポート問い合わせ件数

## 10. 未決事項
- 退会後の再登録ポリシー
- 使い捨てメール対策の要否
