# Feature Spec: Photo RGB Cube Mapping

## 1. 背景

- 画像アップロード後の分析結果は現在 Lab scatter や histogram に分散しており、RGB キューブ上での色分布把握ができない
- IroMap の技術MVPである 3D color visualization と、機能MVPである photo analysis を同一導線で接続したい

## 2. スコープ

### In Scope

- アップロード画像の画素サンプルを RGB キューブ上へ点群として重ねて表示する
- 画像由来の点群を固定グリッド表示とは独立して制御できる
- 未アップロード時は従来の RGB キューブ表示を維持する
- 大きな画像でも描画負荷を抑えるため、サンプリングと量子化で点数を制御する

### Out of Scope

- 画像点群の 3D アニメーション
- 複数画像の同時比較
- 点群からのクラスタ自動抽出
- WebGL への描画基盤移行

## 3. 利用者 / ステークホルダー

- 写真の色構造を 3D 空間で直感的に把握したい撮影者
- 配色の偏りや抜けを RGB 空間で確認したいデザイナー

## 4. ユーザーストーリー

- ユーザーとして、アップロードした画像の色が RGB キューブ内のどこに集中するか見たい。なぜなら色分布の偏りを立体的に把握したいから
- ユーザーとして、従来の基準グリッドと画像由来の点群を切り替えたい。なぜなら色空間全体に対する位置関係と実画像の分布を見分けたいから

## 5. 機能要求

### FR-1: アップロード画像の色サンプルを RGB キューブにマッピングできる

#### 受け入れ条件

- 画像アップロード後、RGB キューブ上に画像由来の色点群が表示される
- 各点は RGB 値に対応した 3D 位置へ投影される
- 透明ピクセルは点群生成対象から除外される

### FR-2: 画像点群の表示モードを切り替えできる

#### 受け入れ条件

- `grid only` / `image only` / `grid + image` の少なくとも 3 モードを選択できる
- 画像未選択時に `image only` または `grid + image` を選んでも表示が破綻しない

### FR-3: 画像点群は分析結果と同じ入力画像に追従する

#### 受け入れ条件

- Photo Analysis で選択中の画像を変更した場合、RGB キューブの点群も同じ画像に更新される
- 画像クリア時は RGB キューブの画像点群も消える

### FR-4: 点群表示は操作性を損なわない範囲で描画負荷を制御できる

#### 受け入れ条件

- 高解像度画像でも、サンプリングや量子化により描画点数が上限制御される
- 画像点群を表示しても、キューブ回転操作が実用上継続可能である
- 同一入力画像に対して、点群生成結果は再現可能である

### FR-5: 画像由来の頻出色を視認しやすく表示できる

#### 受け入れ条件

- 出現頻度の高い色は、低頻度色と比較して視認しやすい見た目で描画される
- 初期実装では、少なくとも alpha または点サイズのいずれかで頻度差を表現できる

## 6. 非機能要求

- NFR-1: 1920x1080 相当画像を対象に、画像選択から RGB キューブ点群表示開始まで 2 秒以内を目安とする
- NFR-2: 点群生成ロジックは既存の photo analysis サンプリング方針と整合する
- NFR-3: 画像データはクライアント内処理とし、外部送信しない

## 7. 業務ルール / 制約

- RGB キューブ点群は、Photo Analysis の入力画像を単一の正とする
- 画像由来点群の色表現は sRGB 前提で扱う
- 初期実装では、画像点群は RGB space 表示を主対象とし、他色空間タブでの意味付けは設計で明示する

## 8. エッジケース

- 単色画像でも 1 箇所へ点群が集中して表示できる
- ほぼ全透明の画像で、有効ピクセル 0 件時の空状態が破綻しない
- 同一色が極端に多い画像でも過度な点数増加が起きない

## 9. 成功指標

- ユーザーが画像の色の偏りを RGB 空間内の位置として説明できる
- 既存の固定グリッド表示と比較しながら、実画像の色分布を同一画面で把握できる

## 10. 未決事項

- 画像点群の頻度表現を alpha のみで始めるか、点サイズも併用するか
- HSL / Lab タブ選択時に画像点群をそのまま投影表示するか、RGB タブ限定にするか
