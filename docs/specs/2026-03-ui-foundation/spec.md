# Feature Spec: UI Foundation (shadcn/ui + Radix)

## 1. 背景

- 既存の初期テンプレートUIを、分析ワークベンチ向けの再利用可能なUI基盤へ置き換える必要がある
- 今後の機能実装で一貫したアクセシビリティとデザイントークン運用を維持したい

## 2. スコープ

### In Scope

- shadcn/ui + Radix を導入し、共通UIコンポーネント基盤を整備する
- トップページを IroMap 向け1ページ構成に置き換える
- ライト/ダークテーマ切替を提供する
- ワークベンチのUX改善要求（テーマ導線、操作アクセシビリティ、情報設計、可視化解釈、状態フィードバック）を仕様化する

### Out of Scope

- RGB Cube / Slice / Photo Analysis の本体実装
- 複数ルートへの画面分割
- デザインシステムの完全固定（ブランド最終調整）

## 3. 利用者 / ステークホルダー

- 写真分析機能を利用するエンドユーザー
- 今後UIを実装する開発者

## 4. ユーザーストーリー

- ユーザーとして、分析UIの全体像を1ページで把握したい。なぜなら機能の理解を素早く行いたいから
- 開発者として、共通コンポーネントを使って機能実装したい。なぜならUI品質と実装速度を両立したいから

## 5. 機能要求

### FR-1: shadcn/ui + Radix のUI基盤を利用できる

#### 受け入れ条件

- Button/Card/Input/Select/Tabs/Separator/Skeleton/Tooltip/Toast を利用できる
- デザイントークンが `globals.css` で定義されている

### FR-2: トップページを IroMap 向け構成へ置き換える

#### 受け入れ条件

- 主要操作（可視化・インスペクタ・分析）へ直接到達できる1ページ構成になっている
- 導線は実装都合ではなくユーザー行動順で配置される
- 既存の create-next-app 初期デザインが表示されない

### FR-3: ライト/ダークテーマを切り替えできる

#### 受け入れ条件

- UI操作でテーマが切り替えられる
- テーマ切替後も主要コンポーネントの可読性が維持される

### FR-4: テーマ切替導線と配色トークンの整合を提供する

#### 受け入れ条件

- 画面上にテーマ切替UIを常時表示し、ユーザーが到達可能である
- ワークベンチ配色がテーマに追従し、ライト/ダーク両方で視認性が維持される
- テーマ切替不能時の状態（初期解決前）でも誤解を生まないラベル/表示を行う

### FR-5: カラー選択導線をポインタ非依存で提供する

#### 受け入れ条件

- RGBキューブ / スライスでキーボード操作または代替UIにより色選択が可能である
- 主要インタラクション要素にフォーカス到達できる
- 支援技術向けに操作意図を説明する属性/ラベルを提供する

### FR-6: 情報設計を利用者向けに最適化する

#### 受け入れ条件

- 画面上に内部要件ID（例: `FR-1 / FR-2`）を表示しない
- 機能カード/セクション見出し/補助テキストを含め、`FR-*` など仕様ID文字列をUIに露出しない
- ヘッダー/セクション名称はユーザー行動ベースで命名する
- 1ページ内で「何を先に操作すべきか」が視覚的に把握できる

### FR-7: 可視化結果の解釈補助を提供する

#### 受け入れ条件

- 散布図とヒストグラムに軸・単位・凡例または同等の解釈補助を表示する
- ツールチップ依存にせず、主要値を常時確認できる
- 分析結果の意味（例: 傾向、偏り）を短文で説明できる

### FR-8: 操作結果の状態フィードバックを強化する

#### 受け入れ条件

- コピー成功/失敗、分析中/完了/失敗が見落としにくいUIで通知される
- 状態変更通知はスクリーンリーダーに伝わる
- 繰り返し操作時に古い通知と新しい通知が識別できる

### FR-9: ヘッダーをコンパクトで機能優先のデザインにする

#### 受け入れ条件

- ヘッダーの縦占有を抑え、ファーストビューで主要ワークベンチ要素が確認できる
- 装飾主導の表現を避け、タイトル・説明・主要操作の優先度が明確である
- モバイル幅でもヘッダーが過剰に高さを消費せず、本文導線を阻害しない

## 6. 非機能要求

- NFR-1: キーボード操作で Select/Tabs/Tooltip の基本操作が可能である
- NFR-2: モバイル幅（390px想定）で主要情報が読める
- NFR-3: テーマ切替前後で主要テキスト/UIのコントラストが極端に低下しない
- NFR-4: 操作結果通知は 1 秒以内に表示され、操作継続を妨げない

## 7. 業務ルール / 制約

- 依存方向 `app -> features -> domain/rendering/lib` の方針を維持する
- 本機能では分析ロジックを追加しない

## 8. エッジケース

- テーマ切替直後に hydration mismatch を発生させない
- プレースホルダUIでもレイアウト崩れを起こさない
- キーボードのみ利用時でも色選択の完了経路が存在する
- 大きな画像分析時でも進行状態が利用者に伝わる

## 9. 成功指標

- 今後の画面実装で共通UIを再利用できる状態になる
- 1ページ上で機能ロードマップを説明できる

## 10. 未決事項

- なし
