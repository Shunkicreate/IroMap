# Docs Overview

`docs/` は IroMap の仕様・設計・技術判断を管理するドキュメントルートです。

## 意図
- 実装前に要求と設計を明確化し、変更理由を追跡可能にする
- `spec -> design -> tasks -> test-cases` の流れを維持する
- 技術判断は ADR として分離して記録する

## フォルダの意味
- `product/`: プロダクト背景、用語、制約
- `architecture/`: システム構造と品質特性
- `adr/`: 技術的な意思決定記録
- `specs/`: 機能単位の仕様セット
- `development/`: ドキュメント運用ルール

## 書き方の基本フォーマット
1. 背景（なぜ必要か）
2. スコープ（どこまで扱うか）
3. 要求 or 設計（何をどう実現するか）
4. 受け入れ条件 / テスト観点（どう検証するか）
5. 未決事項（決まっていない点）
